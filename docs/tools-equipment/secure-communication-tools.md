## Secure Communication Tools

| Tool | Type | Tiers | Notes |
|---|---|---|---|
| **UZH institutional email** | Email | 1–3 | Standard for official communication. Encryption available via S/MIME certificates. Not end-to-end encrypted by default — avoid for Tier 4–5 content in message body. |
| **UZH institutional email + S/MIME** | Encrypted email | 1–5 | With S/MIME certificate enabled, suitable for all tiers. Recipient must also support S/MIME. |
| **ProtonMail** | Encrypted email | 1–5 | Swiss-based (Geneva). End-to-end encryption. Free tier available. Good for narrators who prefer privacy. |
| **Threema** | Encrypted messaging | 1–5 | Swiss-based (Pfäffikon SZ). End-to-end encryption. No phone number required. ~CHF 5 one-time. |
| **Signal** | Encrypted messaging | 1–4 | End-to-end encryption. Free. Open source. Servers in the US but messages are encrypted. Avoid for Tier 5 where Swiss-only infrastructure is preferred. |
| **SWITCHfilesender** | Large file transfer | 1–4 | Available to Swiss universities. Up to 250 GB. Files encrypted in transit. Temporary links. Not designed for highly restricted long-term access. |
| **UZH OneDrive / SharePoint** | File sharing | 1–3 | UZH-provided Microsoft 365. Swiss/EU data centers (verify current policy). For Tiers 4–5: use UZH Research Storage instead. |
